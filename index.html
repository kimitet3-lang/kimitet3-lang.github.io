The smallest stack that can ship

Start with static files and local work. Add a database, email, or model API only when the product actually needs it.

01 / Ship

GitHub + Pages

Version-controlled source and a public static site. No server is required for the first useful version.

  • Root-level index.html
  • GitHub Pages branch deployment
  • Cloudflare DNS only when you own a domain
02 / Compute

Browser + edge

Keep simple features in the browser. Use Cloudflare Workers for small APIs and D1 for small relational data when needed.

  • Workers Free: quota and CPU limits apply
  • D1: small databases, measured usage
  • R2: object storage with free egress
03 / AI

Route by task

Use a local or free-tier model for experiments, then switch providers only when speed, quality, or uptime justifies it.

  • Reasoning: DeepSeek-family models
  • Coding: Qwen Coder-family models
  • General: Llama, Gemma, or Mistral families
Reality check: “Free” means no recurring infrastructure charge within a provider’s limits. It does not mean unlimited traffic, guaranteed uptime, free domains, or free payment processing.

ONE OPERATING LOOP

From request to verified result

Every agentic task follows the same auditable path instead of producing disconnected snippets.

01

Capture

State the goal, constraints, files, and acceptable side effects.

02

Route

Choose the smallest tool or model that can do the job.

03

Build

Change the source of truth, not a pasted copy or an untracked draft.

04

Verify

Run checks, inspect the diff, and test the public result.

05

Log

Record what changed, what was verified, and what remains blocked.
